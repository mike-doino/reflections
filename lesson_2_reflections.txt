What happens when you initialize a repository? Why do you need to do it?

	The files are added to the repository and it prepares to track the changes to all the files when a commit is done.

How is the staging area different from the working directory and the repository? What value do you think it offers?

	The staging area is the temporary holding area between the working directory and the repository. The staging area allows you go group files together before you do a commit.

How can you use the staging area to make sure you have one commit per logical change?

	You can use diff to see what files have been changed.

What are some situations when branches would be helpful in keeping your history organized? How would branches help?

	Branches are great when you want to make a significant change to your code, but still want to keep the original.

How do the diagrams help you visualize the branch structure?

	Diagrams help you see the branches in the git repositories.
